# vatbrain项目冷启动草稿
## 目标
在各个AI厂商提供的API之上构建出统一的抽象层用于LLM推理（例如：文本生成，多模态理解、图片生成、音视频生成、向量化、function-call等等）。
## 设计构想
- 以类似`response api`的形式为基本调用方式，但要求每次推理时传入全量上下文序列以应对`response`失效或厂商仅支持全量`completion`调用的情况。
- 原生支持异步、流式处理。
- 参考目前openai的api设计，以Item作为高层次抽象统一`文本信息`、`function call信息`、`图片`、`视频`、`音频`、`通用文件`、`embeding vector`等需要提供给LLM推理或由LLM生成的上下文组件。
- 每个Item拥有一个类似`role`的属性，指明该Item的用途（或更进一步，在推理上下文中的角色？）。
- 统一流式处理时相关event的设计。
- 统一token统计信息。
- 抽象层是所有AI厂商原生API可实现特性的并集。
	- 不强求每个厂商均适配抽象层的所有特性，提供已支持特性的查询接口。
